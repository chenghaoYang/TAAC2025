04 — SID PREDICTION HEAD (model.py — beamsearch_sid)
Step 1
SID1 Prediction
log_feats → HSTU self-attn
→ LayerNorm
→ Projection(codebook_size)
→ LogSoftmax
→ Top-B beams
(B = beam width, e.g. 5)
Step 2
SID2 Query Construction
For each beam j:
e₁ⱼ = SID_Embedding(s1ⱼ)
q₂ⱼ = Linear(concat(e₁ⱼ, log_feats[:, -1, :]))

Multi-layer cross-attn:
q₂ⱼ → HSTU cross-attn with
all_seq_logfeats as K,V
Step 3
Beam Search Merge
Score(s1ⱼ, sk) =
log P(s1ⱼ) + log P(sk|s1ⱼ)

Top-K₂ (e.g. 384) from
B × codebook_size combos
→ C_sid candidate set
→ reverse lookup → item IDs
SID Training Loss
L_SID1 = CrossEntropy(SID1_logits, y_sid1) — uses log_feats as Q,K,V
L_SID2 = CrossEntropy(SID2_logits, y_sid2) — uses SID1_embedding + log_feats as Q
Teacher forcing: training uses ground-truth SID1 to construct SID2 query
Inference: uses predicted SID1 (beam search) for SID2 query
05 — LOSS FUNCTIONS (utils.py + model.py)
InfoNCE with LogQ Debiasing
L = -log( exp(sim(q,k⁺)/τ - logq(k⁺)) / Σ exp(sim(q,kⱼ)/τ - logq(kⱼ)) )

In-Batch Negatives: other users' pos items
LogQ: log(count(item)/total) correction
Similarity: cosine (F.normalize) > dot product
Temp τ = 0.02 (fixed, better than learnable)

Without LogQ: 0.0378
With LogQ: 0.0949 (+150%)
SID Cross-Entropy Loss
L_SID1 = CE(SID1_logits, y_sid1)
L_SID2 = CE(SID2_logits, y_sid2)

Applied only on loss_mask positions
(where next_token_type == 1)

Monitors: SID Hit@10 for both levels
SID2 easier than SID1 (limited combos)
Total Loss
L_total = L_InfoNCE + α·L_SID1 + β·L_SID2

Optional: Reward Model Loss
→ EnhancedSortMLP(HSTU cross-attn)
→ BCE on CTR label
weighted by cos_sim × p_ctr

+ MoE Aux Loss (if MoE enabled)
+ Weight Decay (1e-5)

08 — DISTRIBUTED TRAINING (dataparallel.py — FedSGD)
FedSGD Architecture (No TCP)
Constraint: TCP/IP disabled, no DDP/DeepSpeed
Solution: Single-process, multi-thread, CUDA-only

1. Master GPU (cuda:0) = Parameter Server
2. Deep-copy model to each GPU
3. Global batch → split → k micro-batches
4. Parallel forward+backward on all GPUs
5. Gather gradients to cuda:0
6. Optimizer step ONLY on cuda:0
7. Broadcast params to all replicas

Optimizer state: 1 copy (not k copies)
Async data transfer: non_blocking=True
Data Preprocessing Pipeline
Offline:
item_exposure_data.py → exposure stats per item
timestamp_buckets.py → 8192 time buckets
preprocess_batch.py → batch pickles

Online (dataset.py):
JSONL + offset pkl → fast random access
LogP computation: log(count/total)
Percentile ranking per bucket
Feature dropout (random zeroing)

PreprocessedDataLoader:
Batch pickles → ThreadPoolExecutor loading
Sliding window prefetch → ordered yield
